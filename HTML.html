HTML（HyperText Markup Language）是一种用于创建和呈现超文本的标记语言。它被用来描述和组织网页的内容和结构，使得文本能够具有交互性和结构化。



HTML的一些关键要素：
标签（Tags）：HTML文档中的基本单位。标签用于指示文档中的不同部分和组件，并告诉浏览器如何处理这些部分。例如，
<p>标签表示段落，
<a>标签表示超链接，
<img>标签表示图像等。
标签的嵌套和文本：HTML标签可以嵌套在其他标签内部，形成层次结构，用于组织文档的结构。同时，标签内部可以包含文本内容，使得文本和标记一起组成文档的内容。例如：

<div>
    <p>This is a paragraph with <em>emphasized</em> text. </p>
</div>

标签属性（Attributes）：HTML标签可以具有属性，用于提供附加信息或控制标签的行为和外观。属性通常以键值对的形式出现，放在标签的开始标记中。例如：

<a href="https://www.example.com">Visit our website</a>

HTML中的一些重要标签包括：

<html>：HTML文档的根元素，所有其他元素都嵌套在<html>标签内部。
<head>：包含了文档的元信息和头部元素，用于描述文档的一些属性和配置，例如文档的标题、字符编码、样式表链接等。<head>标签中常见的子标签包括：
        <title>：定义文档的标题，显示在浏览器的标题栏或标签页上。
        <meta>：定义文档的元数据，如字符编码、关键字、描述等。
        <link>：用于引入外部资源，如样式表、图标等。
<body>：包含了文档的主要内容，是可见部分的主要容器。在<body>标签内部放置的内容会在浏览器中显示出来，包括文本、图像、链接等。通常，网页的主要内容都放在<body>标签内。

在HTML文档中，<head>和<body>是两个重要的部分，它们分别包含了文档的元信息和主要内容。<head>用于描述文档的一些属性和配置，而<body>用于呈现文档的主要内容。 因此，大多数的文档组件（例如段落、标题、图像、链接等）应该放置在<body>标签内，而<head>主要用于配置和描述文档的属性。

HTML文档例子：

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8" />
        <title>A web page</title>
    </head>
    <body>
        <h1>An example web-page</h1>
        <p>A paragraph of text here, perhaps with <a href='./another_page/'>a link</a></p>
    </body>
</html>

<!DOCTYPE html>: 声明文档类型。早期的 HTML（大约 1991-1992 年）文档类型声明类似于链接，规定了 HTML 页面必须遵从的良好规则，能自动检测错误和其他有用的东西。

<meta charset="utf-8">: <meta> 元素。这个元素代表了不能由其他 HTML 元相关元素表示的元数据，比如 <base>、<link>、<script>、<style> 或 <title>。charset 属性将你的文档的字符集设置为 UTF-8，其中包括绝大多数人类书面语言的大多数字符。有了这个设置，页面现在可以处理它可能包含的任何文本内容。没有理由不对它进行设置，它可以帮助避免以后的一些问题。


常用标签（tags）：

结构标签：
<html>：定义 HTML 文档的根元素。
<head>：包含文档的元信息，如标题、字符编码等。
<title>：定义文档的标题，显示在浏览器的标题栏或标签页上。
<body>：包含文档的主要内容，是可见部分的主要容器。
<header>：定义文档的页眉，通常包含网站的标题、标志和导航链接。
<footer>：定义文档的页脚，通常包含版权信息、联系方式等。
<main>：定义文档的主要内容区域。
<nav>(Navigation)：定义导航链接的容器。
<article>：定义独立的、可独立分配的内容，如新闻文章、博客帖子等。
<section>：定义文档的一个区段或部分。

文本标签：
<p>(Paragraph)：定义段落。
<h1>、<h2>、<h3>、<h4>、<h5>、<h6>(Heading)：定义标题，按重要性依次递减。
<span>：定义文本的行内容器，用于设置文本的样式。
<div>(Division)：对文档中的内容进行结构化分组的块级元素。与<span>相比，<div>默认会开始一个新的行，因为它是一个块级元素，而<span>是一个内联元素。
<strong>、<em>(Emphasis)：分别定义强调和重点文本。
<a>(Anchor)：锚点元素，用于定义超链接。
<br>(Break)：定义换行。

列表标签：
<ul>(Unordered List)：定义无序列表。
<ol>(Ordered List)：定义有序列表。
<li>(List Item)：定义列表项。

图像、多媒体标签：
<img>(Image)：定义图像。
<video>：定义视频。
<audio>：定义音频。

表格标签：
<table>：定义表格。
<tr>(Table Row)：定义表格行。
<td>(Table Data Cell)：定义表格数据单元格。


常见的属性（attributes）：

id：为元素指定唯一的标识符，用于在 CSS 和 JavaScript 中引用该元素。
class：为元素指定一个或多个类名，用于定义元素的样式或者在 JavaScript 中选择该元素。
src：指定嵌入的资源的 URL，如图像、音频、视频等。
href：指定链接的目标 URL，用于创建超链接。
alt：指定图像的替代文本，用于图像加载失败时显示的文本或者屏幕阅读器的内容。
title：为元素提供额外的描述性信息，通常在用户将鼠标悬停在元素上时显示为工具提示。
style：为元素指定内联样式，用于直接定义元素的样式。
width、height：指定图像、表格等元素的宽度和高度。
type：用于指定输入元素的类型，如 <input type="text">、<input type="checkbox"> 等。
placeholder：指定输入字段的占位符文本，用于提供对用户所需输入的提示。
name：为表单元素指定名称，用于在提交表单时将值与表单控件关联起来。
value：为表单元素指定初始值。
checked：用于指定复选框或单选按钮的默认选中状态。
disabled：禁用元素，使其不可编辑或不可交互。


常见的输入（input）类型：

  • E-mail地址字段：将 type 属性设置为 email （type="email"）就可以使用这种控件。用户需要输入一个合法的电子邮件地址，任何其他输入都会使得浏览器在表单提交时显示错误信息。
  • 查询字段（Search fields）；旨在用于在页面和应用程序上创建搜索框。将 type 属性设置为 search 就可以使用这种控件。
  • 电话号码字段：在 type 属性中使用 tel 值，即可创建一个专门用于输入电话号码的文本域。
  • URL 字段：在 type 属性值中使用 url，即可创建一个用于输入网址的文本字段。
  • 数字字段：用于输入数字的控件可以由 type 为 number 的 <input> 元素创建。这个控件外观与文本域类似，但只允许浮点数输入，并通常以旋转器（spinner）的形式提供按钮来增加和减少控件的值。在有动态键盘的设备上，一般会显示数字键盘。
  • 滑块控件：在 <input> 元素中使用 range 作为属性 type 的值，就可以创建一个滑块，滑块可以通过鼠标、触摸，或用键盘的方向键移动。
  • 日期和时间选择器：日期和时间控件可由 <input> 元素和一个合适的 type 属性值来创建，该值取决于要收集的类型（日期、时间、还是以上全部）。
  • 颜色选择器控件：用于输入颜色的控件可以由 type 为 color 的 <input> 元素创建

客户端验证（Client-side validation）：正如你在上面看到的，email 与其他较新的 input 类型一样，提供了内置的客户端错误验证，在数据被发送到服务器之前由浏览器执行。 它是引导用户准确填写表格的一个有用的辅助工具，可以节省时间：可以立即知道你的数据是否正确，而不需要等待服务器返回结果。
但它不应该被认为是一种详尽的安全措施！你的应用程序始终应该在服务器端和客户端对任何表单提交的数据进行安全检查，因为客户端验证太容易被关闭了，所以恶意用户仍然可以很容易地将坏数据发送到你的服务器。

模板化（Templating）

本周workshop的应用程序是一个包含学生、单元和成绩的极简大学数据库。模型文件夹中的数据库接口与学生类和单元类一起构成了数据模型--你可以搜索数据库中的所有学生或所有单元。当然，在真正的数据库中，您还可以通过 ID 搜索单个学生。主页面将以 HTML 文件的形式从 src/main/resources/web 的类路径中提供。

文件 Templates.java 设置了 thymeleaf 模板引擎。@Component 注解告诉 Spring 来管理这个类；其他需要模板的类可以通过用 @Autowired 声明一个正确类的字段来请求模板，就像你在 Controller.java 中看到的那样。
